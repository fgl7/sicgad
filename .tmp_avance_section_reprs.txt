2: 'En el marco de la propuesta metodológica para medir rendimiento productivo y eficiencia de procesos (plantas PCS, KCl y Li2CO3), se inició la construcción del componente de “Desempeño” dentro de SICGAD. El objetivo de este avance es que el sistema no solo reciba y valide datos, sino que también pueda transformar esa información en indicadores mensuales comparables, con trazabilidad y posibilidad de actualización cuando cambien los esquemas de datos.'
3: ''
4: 'Principales mejoras implementadas hasta el momento:'
5: ''
6: '- Se incorporó una nueva sección “Desempeño” en la interfaz (visible para el usuario Administrador del sistema) para centralizar el trabajo relacionado con rendimiento y eficiencia.'
7: '- Se implementó un enfoque flexible de “asignación de variables”, donde el Administrador puede vincular las variables de las fórmulas (por ejemplo Msales(m), Msalmuera(m-Δt), Xsolids(m-Δt)) con campos existentes en los esquemas diarios o mensuales de la planta. Esto permite adaptar el cálculo sin necesidad de modificar el código cuando se actualizan nombres o estructuras de los esquemas.'
8: '- Se preparó el sistema para manejar resultados en dos estados:'
9: '  - “Draft”: resultados preliminares a partir de datos cargados/publicados durante el mes (sin esperar el cierre mensual).'
10: '  - “Certificado”: resultados a partir de datos mensuales certificados y aprobados, usados como referencia oficial.'
11: '- Se desarrolló un primer MVP para la Planta de Concentración de Sales (PCS) enfocado en la Fórmula 1 (Rendimiento de Producción Mensual %). En esta pantalla el Administrador:'
12: '  - Selecciona el mes a evaluar.'
13: '  - Asigna las variables requeridas por la fórmula a campos reales del sistema (por planta).'
14: '  - Visualiza un resultado estimado (preview) para Draft y Certificado.'
15: '  - Cuenta con un gráfico inicial para ver la tendencia mensual.'
16: '- Se sentaron bases para el “catálogo” de variables e indicadores, de modo que el sistema pueda ir incorporando progresivamente las demás fórmulas del documento (recuperación, energía, balances, pérdidas, etc.) manteniendo una lógica uniforme.'
17: '- Se ajustó la gestión de dependencias para asegurar compatibilidad con carga y lectura de archivos Excel, ya que SICGAD trabaja con plantillas de carga y materialización de datos.'
18: ''
19: 'Este avance deja listo el camino para la siguiente etapa: alimentar la Fórmula 1 con datos reales (tanto diarios como mensuales certificados), ajustar el criterio de cálculo (por ejemplo sumas por día, selección de columnas correctas y desfase Δt) y extender el mismo mecanismo a las siguientes fórmulas de desempeño.'
