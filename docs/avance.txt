AVANCE SICGAD (AL MOMENTO)
Alcance de este documento: registro de esquemas de produccion (diario) y certificacion mensual.


1) SISTEMAS CONCLUIDOS Y PROBADOS (PRODUCCION DIARIA Y CERTIFICACION MENSUAL)

1.1 Autenticacion y control de accesos (base del sistema)

- El sistema opera con inicio de sesion restringido a usuarios registrados.
- Se implemento una navegacion (sidebar) por rol para reducir errores operativos y evitar accesos a pantallas no relevantes.
- Roles reconocidos actualmente:
  - SUPERADMIN (tecnico, creado con createsuperuser): administracion del Django admin y configuracion tecnica.
  - ADMIN (rol del sistema): gestiona usuarios, esquemas, proyectos y configuraciones funcionales.
  - LOADER (cargador): realiza cargas de datos (borradores) y envio a validacion.
  - VALIDATOR (validador): revisa y decide (aprobar/rechazar) segun reglas y niveles.
  - VIEWER (visualizador): acceso a vistas permitidas (principalmente datos publicados).
- El alcance de permisos puede definirse como global o por planta/proyecto, segun membership.

1.2 Esquemas (tablas flexibles) para produccion y certificacion

- Se consolidaron componentes para crear y administrar "esquemas" (DatasetType) con sus columnas (ColumnDef).
- Los esquemas operan como tablas flexibles: el sistema valida estructura a partir del esquema, y materializa datos publicados en un modelo por celda (PublishedDataPoint) para consulta y auditoria.
- Se soportan periodos por dataset y se establece periodicidad (diaria, semanal, mensual o flexible) como parte del esquema.
- Se habilito un flujo de aprobacion de esquemas (borrador -> pendiente -> aprobado/rechazado) para asegurar que solo esquemas aprobados sean usados en carga.

1.3 Flujo de carga de datos y estados (borrador -> validacion -> publicado)

- Se concluyo el flujo base de carga de datos por archivo:
  - Cargar archivo (CSV/Excel) con periodo y esquema aprobado.
  - Crear instancia en estado borrador (DatasetInstance).
  - Revisar y corregir (si esta en borrador).
  - Enviar a validacion.
- Se concluyo el soporte de captura manual para ciertos escenarios, generando tambien instancias en estado borrador.
- Se habilito historial de cargas y detalle de instancia, para rastrear lo cargado, estado, errores y acciones.

1.4 Certificacion mensual (produccion) y trazabilidad

- Se concluyo el flujo de certificacion mensual de produccion, apoyado en:
  - Reglas de periodicidad y periodos del mes.
  - Revision y correccion con trazabilidad (cuando corresponde).
  - Publicacion final segun reglas definidas de aprobacion.
- Se reforzo la trazabilidad con registro de auditoria (AuditLog) y registro de decisiones/acciones de validacion.

1.5 Controles de seguridad y calidad de datos (aplicados al flujo de produccion/certificacion)

- Restriccion de acciones por estado: solo borradores pueden corregirse/eliminarse.
- Separacion clara entre datos borrador y datos publicados:
  - Los validadores y participantes del flujo pueden revisar borradores.
  - Los datos oficiales se visualizan desde instancias publicadas/materializadas.
- Auditoria de acciones: carga, correccion, envio a validacion, validacion, publicacion y eliminacion, con detalle del objeto afectado.


2) PRUEBAS (PRODUCCION DIARIA Y CERTIFICACION MENSUAL)

2.1 Enfoque de pruebas

- Se realizaron pruebas funcionales iterativas con datos reales y casos representativos:
  - Carga por archivo y captura manual.
  - Correcciones en borrador y re-envio a validacion.
  - Aprobacion y rechazo por validadores.
  - Publicacion y verificacion de lectura de datos publicados.
  - Certificacion mensual y consistencia entre flujos.
- Cada prueba fue acompanada de ajustes puntuales cuando se encontraron problemas de:
  - Estructura del esquema (nombres de columnas, orden, tipos).
  - Comportamiento del flujo (permisos por rol, alcance por planta/proyecto, estados).
  - Visualizacion (lectura de datos publicados vs borradores).

2.2 Resultados de pruebas (estado actual)

- El flujo de produccion diaria y certificacion mensual se considera implementado y probado a nivel funcional.
- Se confirmo que el sistema:
  - Soporta multiples usuarios con roles distintos y alcances distintos.
  - Permite registrar esquemas, cargar datos, validar, publicar y auditar acciones.
  - Mantiene trazabilidad y control de estados para evitar modificaciones fuera de flujo.

2.3 Pendientes recomendados (para este alcance)

- Consolidar un set de escenarios de prueba documentados (casos base y casos borde) con evidencia (capturas/logs).
- Preparar un entorno piloto con datos reales y monitoreo de incidencias, antes de despliegue productivo.
- Documentacion de usuario por rol (cargador/validador/administrador) centrada en produccion y certificacion mensual.


3) FLEXIBILIDAD DEL SISTEMA Y NUEVAS INCORPORACIONES EN PRUEBA (UPSTREAM RECURSOS EVAPORITICOS)

3.1 Extensibilidad de esquemas (mas alla de produccion)

- El sistema de esquemas es lo suficientemente flexible para incorporar otros tipos de datos ademas de produccion:
  - Esquemas de inversiones (ejecucion financiera/fisica, curvas, reportes).
  - Esquemas estaticos o de carga unica (por ejemplo proyecciones o valores iniciales).
- Se realizaron pruebas de ingreso de esquemas no-productivos y funcionaron correctamente en cuanto a:
  - Registro de esquema y columnas.
  - Carga y control de estados.
  - Revision/visualizacion segun permisos.

3.2 Esquemas estaticos y control de cambios con justificacion (en pruebas)

- Se incorporo el concepto de "carga unica" por esquema para proteger informacion estatica (ej. proyecciones o valores base).
- La logica busca asegurar que cualquier modificacion posterior quede justificada y registrada en el sistema.
- Este control se encuentra en proceso de pruebas y ajuste, especialmente para escenarios donde ya existian cargas previas.

3.3 Registro y reporte de proyectos (inversiones) para upstream (en pruebas)

- Se implemento un modulo de Proyectos para registrar proyectos y asociarlos a multiples plantas/unidades operativas.
- Se implemento un sistema de configuracion de reportes por proyecto y una vista de reporte con:
  - Datos de resumen del proyecto.
  - Curva S programada y ejecutada.
  - Filtros (gestion, fuente publicados/borradores y seleccion semanal cuando aplica).
- Estas capacidades se estan probando actualmente para proyectos ejecutados en el upstream del sector de recursos evaporiticos (exploracion y explotacion).

3.4 Estado actual

- A la fecha continun las pruebas y ajustes de estas nuevas incorporaciones (inversiones/proyecciones/proyectos) para integrarlas de forma consistente al SICGAD, manteniendo:
  - Seguridad por rol y alcance.
  - Control de cambios con justificacion cuando corresponde.
  - Trazabilidad y auditoria de acciones.
