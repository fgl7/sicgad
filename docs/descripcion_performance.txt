3.1 Avance del módulo de fórmulas de desempeño

En el marco de la evolución funcional del SICGAD, se ha desarrollado el módulo de fórmulas de desempeño como un componente orientado a transformar datos validados en indicadores calculados con utilidad analítica e institucional. Este avance responde a una necesidad concreta: no basta con registrar y publicar información, sino que también es necesario disponer de mecanismos que permitan interpretarla, relacionarla y convertirla en medidas de rendimiento, eficiencia, avance o comportamiento de variables relevantes para la gestión.

Tradicionalmente, este tipo de cálculos suele realizarse fuera de los sistemas institucionales, mediante hojas de cálculo separadas, archivos personales o fórmulas construidas manualmente por distintos usuarios. Esa situación genera varios problemas: dependencia de personas específicas, dificultad para replicar el cálculo de manera consistente, escasa trazabilidad sobre la metodología aplicada y riesgo de trabajar con datos no oficiales o desactualizados. Frente a ello, el módulo de fórmulas de desempeño del SICGAD representa un avance significativo al incorporar dentro del propio sistema la capacidad de definir, calcular, revisar y conservar indicadores derivados sobre una base formal y controlada.

3.1.1 Finalidad del módulo

El objetivo principal de este módulo es permitir la construcción de indicadores de desempeño a partir de datos que ya han sido validados y publicados en el flujo ordinario del SICGAD. En lugar de depender de cálculos externos o informales, el sistema ofrece un entorno institucional donde las fórmulas pueden ser definidas, revisadas y aplicadas directamente sobre información oficial.

En este sentido, el módulo permite:

- aprovechar datos ya publicados como base para nuevos indicadores;
- construir relaciones entre variables de manera ordenada y reutilizable;
- reducir la dependencia de hojas de cálculo o procesos externos;
- fortalecer la capacidad analítica del sistema sobre información oficial.

3.1.2 Problemática que atiende

El desarrollo de este módulo permite atender de manera progresiva las siguientes limitaciones:

- realización de cálculos manuales fuera del sistema;
- falta de estandarización en la construcción de indicadores;
- baja trazabilidad sobre cómo se obtiene un resultado analítico;
- riesgo de usar información no validada como base de cálculo;
- dificultad para repetir o actualizar indicadores de forma consistente.

En consecuencia, el módulo contribuye a institucionalizar el cálculo de indicadores, de modo que estos dejen de depender de prácticas aisladas y pasen a formar parte del ecosistema formal de gestión de datos.

3.1.3 Lógica general de funcionamiento

La lógica del módulo parte de un principio central: un indicador de desempeño debe construirse sobre datos confiables y previamente publicados. Por ello, las fórmulas no se calculan sobre información provisional, sino sobre datos que ya han atravesado el flujo de carga, validación y publicación del SICGAD.

De manera general, el proceso funciona de la siguiente forma:

- se selecciona una entidad sobre la cual se desea construir el indicador;
- se eligen variables disponibles desde esquemas aprobados y activos;
- se define una expresión matemática que relaciona dichas variables;
- el sistema genera una vista preliminar del resultado;
- una vez aprobada la fórmula, el resultado puede materializarse como un dataset derivado;
- el indicador queda disponible para consulta y reutilización dentro del sistema.

Esta lógica convierte al módulo en una extensión natural del flujo institucional del SICGAD, ya que toma datos oficiales y los transforma en información analítica también formalizada.

3.1.4 Avance funcional del módulo

3.1.4.1 Selección flexible de variables

Uno de los avances más relevantes del módulo es la flexibilidad para elegir variables de cálculo. El sistema no obliga a trabajar con un conjunto rígido de indicadores predefinidos, sino que permite seleccionar distintas columnas numéricas provenientes de esquemas aprobados y activos de una entidad.

Este enfoque ofrece las siguientes ventajas:

- adaptación a diferentes necesidades de gestión y análisis;
- posibilidad de construir indicadores específicos según el problema institucional;
- aprovechamiento de datos ya disponibles dentro del sistema;
- mayor autonomía para el diseño de indicadores sin requerir desarrollos separados.

En términos prácticos, si existen variables relacionadas con producción, costos, volúmenes, rendimiento, avance u otras magnitudes cuantificables, estas pueden convertirse en insumos para el cálculo de nuevos indicadores.

3.1.4.2 Construcción de fórmulas con expresiones definidas por el usuario

El módulo permite definir fórmulas de manera clara mediante aliases o referencias simples, lo que facilita su construcción y comprensión. En lugar de trabajar con expresiones complejas basadas en nombres extensos de columnas, el sistema permite componer la fórmula usando variables identificadas de manera sencilla.

Este mecanismo aporta:

- mayor claridad en la definición del cálculo;
- facilidad de lectura y revisión de la fórmula;
- menor riesgo de errores durante la configuración;
- mayor accesibilidad para usuarios responsables del análisis.

3.1.4.3 Validación de consistencia

La flexibilidad del módulo se complementa con reglas de control que preservan la consistencia técnica del indicador. Una de las validaciones más importantes es la relacionada con la periodicidad de las variables. El sistema evita combinar libremente datos que pertenecen a frecuencias incompatibles, como registros diarios y mensuales dentro de una misma fórmula.

Este control permite:

- proteger la calidad metodológica del indicador;
- evitar resultados técnicamente inconsistentes;
- asegurar que la interpretación del resultado sea válida;
- mantener coherencia entre la fórmula y la naturaleza de los datos utilizados.

3.1.4.4 Cálculo sobre períodos comunes

Otro aspecto relevante del avance funcional consiste en que el sistema calcula resultados sobre períodos comunes con datos completos. Esto significa que no se generan indicadores a partir de cruces incompletos o de información parcial, sino sobre la intersección real donde todas las variables requeridas disponen de datos publicados.

Este criterio aporta:

- mayor consistencia en el resultado final;
- reducción del riesgo de cálculos sobre vacíos o faltantes;
- mejor interpretación de la serie resultante;
- mayor confianza en el uso posterior del indicador.

3.1.4.5 Vista preliminar o preview

Antes de aprobar una fórmula, el sistema permite visualizar un adelanto de sus resultados mediante un gráfico y una tabla compacta. Esta funcionalidad constituye una etapa de comprobación previa, orientada a verificar si el comportamiento del indicador es coherente con la lógica esperada.

El preview resulta valioso porque:

- permite revisar la fórmula antes de su aprobación definitiva;
- ayuda a identificar inconsistencias o comportamientos no esperados;
- ofrece una comprobación visual del resultado;
- fortalece la confianza del usuario en el cálculo realizado.

3.1.4.6 Materialización del indicador

Una vez aprobada la fórmula, el sistema puede materializar el resultado como un dataset derivado propio del SICGAD. Este aspecto representa un avance institucional importante, porque el indicador deja de ser un cálculo temporal y pasa a formar parte del patrimonio informativo del sistema.

La materialización permite:

- conservar resultados por período dentro del entorno institucional;
- reutilizar el indicador en consultas y análisis posteriores;
- evitar recálculos manuales repetitivos;
- consolidar el indicador como información derivada formal y trazable.

3.1.4.7 Recálculo sobre nueva información publicada

El módulo también contempla la posibilidad de recalcular fórmulas aprobadas cuando se publican nuevos datos de origen. Esto implica que el indicador no queda congelado, sino que puede evolucionar de acuerdo con la actualización de la base oficial del sistema.

Este avance contribuye a:

- mantener vigencia en los resultados;
- reducir trabajo manual de actualización;
- asegurar consistencia entre datos base e indicador derivado;
- fortalecer el uso continuo del módulo en la operación institucional.

3.1.5 Valor institucional del avance alcanzado

Desde una perspectiva institucional, el valor principal del módulo radica en que combina flexibilidad y control. Es flexible porque permite elegir variables y construir fórmulas adaptadas a necesidades reales. Pero al mismo tiempo es controlado, porque trabaja sobre datos validados, aplica reglas de consistencia, ofrece una etapa previa de revisión y conserva trazabilidad sobre el resultado final.

El módulo fortalece la capacidad del SICGAD para:

- transformar datos operativos en conocimiento útil;
- construir indicadores de rendimiento, eficiencia, avance o relación entre variables;
- reducir la dependencia de procesos analíticos externos;
- institucionalizar metodologías de cálculo dentro de una plataforma común;
- ampliar el valor de la información ya cargada, validada y publicada.

En ese sentido, este avance proyecta al SICGAD no solo como una herramienta de registro y validación, sino también como una plataforma capaz de apoyar procesos de interpretación y medición del desempeño institucional.

3.1.6 Estado actual

En síntesis, el módulo de fórmulas de desempeño constituye un avance funcional relevante dentro del SICGAD, al permitir la construcción de indicadores calculados de manera ordenada, trazable y reutilizable. Gracias a la flexibilidad para seleccionar variables, a la posibilidad de definir expresiones propias y a la materialización de resultados como datasets derivados, el sistema amplía de manera significativa su capacidad analítica y fortalece su utilidad institucional como herramienta integral de gestión, validación, análisis y seguimiento de información.
