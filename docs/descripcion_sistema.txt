3.1 Avance del sistema

En el marco de una demostración institucional para la Agencia Boliviana de Energía Nuclear (ABEN), el Sistema Integral de Carga, Gestión, Validación y Análisis de Datos Institucionales (SICGAD) se presenta como una plataforma orientada a centralizar, ordenar, validar, resguardar y poner a disposición información oficial para seguimiento, análisis y toma de decisiones ejecutivas.

La demostración muestra cómo SICGAD puede apoyar a la Dirección General Ejecutiva de la ABEN en el seguimiento de medicina nuclear y radioterapia, investigación y desarrollo nuclear, radiofarmacia, irradiación multipropósito, mantenimiento de equipos críticos, proyectos estratégicos, seguridad radiológica, calidad y cumplimiento.

3.1.1 Problemática que atiende el sistema

SICGAD atiende problemas frecuentes de gestión institucional:

- dispersión de información en archivos, correos y repositorios aislados;
- dependencia de gestiones manuales para solicitar, recibir y consolidar datos;
- dificultad para identificar la versión oficial y vigente de la información;
- baja trazabilidad sobre cargas, revisiones, aprobaciones y cambios;
- limitada disponibilidad de indicadores oportunos para autoridades y equipos técnicos;
- riesgo de pérdida de continuidad operativa ante cambios de personal.

3.1.2 Lógica general de funcionamiento

El sistema se organiza sobre una estructura institucional de sector, subsector, categoría y entidad. Sobre esta estructura se asignan usuarios, se delimitan alcances, se vinculan esquemas de datos y se controla el acceso a la información.

La secuencia operativa es:

- organización de estructura institucional y entidades responsables;
- creación de usuarios con perfiles y alcances diferenciados;
- definición de esquemas de datos según la necesidad institucional;
- carga de información periódica, histórica o manual;
- revisión y validación institucional de registros;
- publicación de información oficial para visualización, consulta, descarga y análisis;
- trazabilidad y auditoría de acciones relevantes.

3.1.3 Avance por módulos

3.1.3.1 Gestión de usuarios y accesos

El sistema contempla perfiles de administración, carga, validación y visualización, cada uno con permisos y alcances específicos. Esto fortalece el control interno, evita accesos no pertinentes y permite que cada actor interactúe con las funciones y datos que le corresponden.

3.1.3.2 Estructura institucional

SICGAD permite representar la organización institucional mediante niveles jerárquicos y entidades responsables. Esta base ordena responsabilidades, define alcances de usuarios y vincula correctamente cargas, esquemas y consultas.

3.1.3.3 Esquemas de datos

El sistema permite definir estructuras flexibles para distintos tipos de información institucional. Los esquemas aprobados establecen columnas, tipos de dato, periodicidades y flujos de validación.

3.1.3.4 Carga de información

SICGAD admite cargas históricas, periódicas y manuales. El uso de plantillas y validaciones de formato reduce errores y mejora la consistencia de los registros.

3.1.3.5 Validación

La información cargada no se publica automáticamente. Debe ser revisada y aprobada por responsables definidos, garantizando que los indicadores y visualizaciones se alimenten de registros verificados.

3.1.3.6 Visualización, consulta e indicadores

La información validada se transforma en tablas, gráficos, indicadores y mecanismos de descarga según el perfil y alcance de cada usuario. Para la demo ABEN se priorizan KPIs ejecutivos de servicios nucleares, mantenimiento, proyectos y cumplimiento.

3.1.3.7 Auditoría y trazabilidad

El sistema registra acciones relevantes como creación, modificación, envío, aprobación o rechazo de información y configuraciones. Esta capacidad permite reconstruir el historial institucional del dato.

3.1.3.8 Proyectos y convenios

El módulo de proyectos permite registrar iniciativas estratégicas, vincularlas con información validada y habilitar reportes ejecutivos para seguimiento físico, financiero y operativo.

3.1.4 Valor institucional

SICGAD permite pasar de procesos manuales, datos dispersos y reportes tardíos a una gestión institucional digital, trazable, validada y orientada a decisiones ejecutivas.

En la demostración ABEN, todo dato es ficticio y tiene fines exclusivamente demostrativos.
