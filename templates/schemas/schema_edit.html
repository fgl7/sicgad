¿Qué significa cada columna del esquema?

Esta tabla define la estructura de los datos que se van a cargar. Una buena definición aquí evita muchos errores al momento de subir archivos.

  • Nombre: identificador interno del campo (sin espacios). Es el nombre técnico y también el encabezado que se usará en las plantillas y archivos de carga/exportación.
  • Etiqueta: descripción para otros usuarios; explica qué representa la columna y qué tipo de datos se espera.
  • Tipo: tipo de dato esperado (entero, decimal, texto, fecha, sí/no, etc.).
  • Req.: indica si el campo es obligatorio; si está marcado, no se aceptan filas vacías en esa columna.
  • Mín. / Máx.: valores mínimo y máximo permitidos para datos numéricos (ayudan a detectar errores de dedo).
  • Unidad: unidad de medida que se mostrará en pantallas y gráficos (por ejemplo TM/día, m³, %, kWh).
  • Rol gráfico: cómo se usa el campo en los gráficos (eje X, eje Y, serie o filtro).
  • Agreg.: forma en que se resume el campo al agrupar datos (suma, promedio, máximo, mínimo, etc.).
  • KPI: marca este campo como indicador principal para que aparezca destacado en los tableros.
  • Orden: posición en la que se mostrará la columna en tablas y formularios.
  • Activo: indica si la columna sigue en uso. Se puede desactivar sin borrar el historial ya cargado.
  • Eliminar: opción para quitar la columna de este esquema (no elimina los datos ya registrados en versiones anteriores).
